# Fix Evidence Bundles

DevLoop writes a fix evidence bundle for every automated run that can change, validate, or review code. Bundles are designed to make an AI-generated PR auditable without relying on terminal scrollback.

Default location:

```text
.devloop/evidence/<run-id>/
  evidence.json
  diagnosis.json
  firewall-report.json
  patch.diff
  test-before.log
  test-after.log
  sandbox.json
  pr-body.md
  metadata.json
```

## What Is Captured

`evidence.json` is the canonical record. It includes:

- schema version and run id
- DevLoop version and trigger type
- model and provider metadata
- sandbox runner, network mode, timeout, and secret-mount status
- diagnosis summary, confidence, and likely files
- changed files, line counts, patch risk level, and SHA-256 hash
- validation commands and log hashes
- Agent Firewall decision and finding count
- safety flags such as forbidden files, workflow permission edits, secret detection, and test deletion

Logs and PR bodies are redacted before they are written. Raw secrets should never be stored in evidence files.

## CLI

Show a bundle:

```bash
devloop evidence show <run-id> --repo .
```

Verify a bundle:

```bash
devloop evidence verify .devloop/evidence/<run-id>
```

Verification checks:

- required files exist
- `evidence.json` matches the expected schema shape
- `patch.diff` matches the recorded SHA-256 hash
- validation logs match the recorded SHA-256 hashes

Invalid bundles exit with a non-zero status.

Export a bundle:

```bash
devloop evidence export <run-id> --repo . --format markdown --output evidence-artifacts
devloop evidence export <run-id> --repo . --format zip --output evidence-artifacts
devloop evidence export <run-id> --repo . --format json --output evidence-artifacts
```

Zip exports are named `devloop-evidence-<run-id>.zip`.

## Pull Requests

Every PR body generated by DevLoop includes:

```text
## Evidence Bundle

- Run ID
- Risk level
- Sandbox mode
- Test command
- Evidence file path or artifact link
- Human review required
```

GitHub App mode also posts a short evidence summary comment after opening the PR when possible.

## Integrations

Evidence bundles are generated by:

- `devloop autofix`
- `devloop security-autofix`
- GitHub App autofix jobs
- transport-agnostic autofix tools used by future MCP servers
- Agent Firewall repository scans
- FixBench benchmark cases

FixBench stores bundles under each case output directory so benchmark results can be reproduced after temporary work directories are cleaned up.
