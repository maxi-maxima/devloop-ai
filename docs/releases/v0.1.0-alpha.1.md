# DevLoop AI v0.1.0-alpha.1

DevLoop AI is an open-source firewall and autofix agent for AI coding agents. It turns failed CI and security alerts into tested pull requests, with sandboxing, prompt-injection defense, secret redaction, patch review, and evidence bundles.

This alpha supersedes `v0.1.0-alpha.0` as the recommended public alpha release. The `v0.1.0-alpha.0` tag was created before the final launch-readiness documentation commit, so `v0.1.0-alpha.1` realigns the release tag, GitHub prerelease notes, README, checklist, and hardening report.

## What Changed Since v0.1.0-alpha.0

- Updated the public alpha checklist to reflect the published prerelease and immediate post-public protection steps.
- Updated the GitHub hardening report with the final launch gate status.
- Updated package metadata from `0.1.0-alpha.0` to `0.1.0-alpha.1`.

No runtime product code changed from `v0.1.0-alpha.0`.

## What Is Included

- CI AutoFix for failed test logs.
- Security Autofix from SARIF 2.1.0 files.
- GitHub App mode with slash commands.
- GitHub Action template for failed workflow autofix.
- MCP stdio server for compatible coding agents.
- Codex Skill package for reusable autofix workflows.
- Agent Firewall for prompt injection, secret exfiltration, dangerous commands, unsafe patches, and malicious repo instructions.
- Evidence Bundles for auditability.
- FixBench and FirewallBench benchmark suites.
- Local and Docker sandbox runner abstractions.

## Quickstart

```bash
git clone https://github.com/maxi-maxima/devloop-ai.git
cd devloop-ai
npm install
npm run build
./scripts/demo-autofix.sh
```

Run a local diagnosis or dry-run fix:

```bash
devloop diagnose --repo . --log-file ./ci.log
devloop autofix --repo . --log-file ./ci.log --test-command "npm test" --dry-run
```

## Safety Model

DevLoop is conservative by design:

- no auto-merge,
- dry-run defaults where writes are risky,
- human review required before merging,
- forbidden edits to secrets and environment files,
- prompt-injection and malicious instruction checks,
- secret redaction before logs enter evidence bundles,
- patch safety review before applying changes,
- sandboxed validation commands where configured,
- evidence bundles for diagnosis, patch, tests, firewall report, and metadata.

## Known Limitations

- Alpha quality: APIs, CLI output, and config files may change.
- Human review is required for every PR.
- No auto-merge.
- Some fixes will fail or need manual follow-up.
- Docker is recommended for stronger sandboxing.
- Public repository support is the safest path; private repos require careful token and GitHub App configuration.
- Security Autofix handles safe educational patterns first and returns `DEVLOOP_CANNOT_FIX_SAFELY` when the fix is unclear.
- npm publishing is intentionally deferred for this alpha.

## Report Bugs

Open an issue with:

- DevLoop version,
- operating system and Node.js version,
- command run,
- redacted CI log or SARIF excerpt,
- expected behavior,
- actual behavior,
- generated evidence bundle path if available.

Use the issue templates for bug reports, failed CI cases, and security autofix cases.

## Contribute Benchmark Cases

FixBench and FirewallBench are designed to grow with real-world failures. To contribute a case:

1. add a small safe fixture under `benchmarks/fixbench/` or `benchmarks/firewallbench/`,
2. include metadata, expected command, expected decision or success condition, and allowed files,
3. avoid secrets, exploit payloads, and network dependencies,
4. run the relevant benchmark command,
5. open a PR with the generated report summary.
